# End to End Ticketing Lifecycle | Ticketing Workflow Simulation

### 1. Ticket Creation (Intake Phase)
A support ticket is initiated by an end user via a help desk portal or email.

**Captured Information:**
- Issue description (e.g., *"Laptop crashing"*)
- Help topic/category (hardware, software, network, etc.)
- Priority level (Low, Medium, High)

This step ensures proper classification and routing of the issue.

---

### 2. Ticket Triage, Assignment & SLA Setup
Once created, the ticket is reviewed and assigned:

- Routed to the appropriate support team or agent
- Assigned a severity level:
  - Sev-A (High)
  - Sev-B (Medium)
  - Sev-C (Low)
- Matched with an SLA defining response and resolution time

**Purpose:**
- Establish accountability  
- Ensure timely handling of requests  

---

### 3. Ticket Handling & Investigation
The assigned agent begins working the ticket:

- Reviews ticket details and history  
- Communicates with the user for troubleshooting  
- Documents progress using internal notes  
  - Example: *"Waiting on vendor response"*  

**Status Flow:**

---

### 4. Resolution
After identifying a solution:

- The agent applies the fix  
- Sends a resolution summary to the user  
- Updates ticket status to 'Resolved'


---

### 5. Closure
Final stage of the lifecycle:

- Ticket is officially closed (manual or automatic)  
- Final notification sent to the user  
- Ticket is archived for:
  - Reporting
  - Auditing
  - Knowledge base reference
